RENOLIE — Trådstatus (belastning) — Addendum v1.0.7

Formål: At definere præcist, hvordan “Trådbelastning 0–100%” beregnes i hver tråd, så vi ved, hvornår der bør skiftes til ny tråd.

Definition

Trådbelastning = (estimerede tokens i hele tråden indtil nu) / (model-kontekstkapacitet) × 100%.

Kapacitetsgrænser (Standard)

  • Warn@70%: Assistenten viser tydelig advarsel (risiko for træge svar).
  • Hard@85%: Foreslå trådskifte aktivt og lever færdig handover-tekst.
  • Cut@95%: Skift tråd omgående for at undgå klip/tab.

Estimeringsmetode (praktisk)

  1. Tekst → tokens: antag ca. 1 token ≈ 4 tegn (ASCII-baseret estimering).
  2. Code blocks og filer tæller fuldt ud (inkl. whitespace).
  3. Billeder og binære filer ignoreres; deres referencer tæller som tekst.
  4. System + projekthukommelse: lægges oveni med fast overhead på ~3.5k tokens.
  5. Aktuel model-kontekst: ~128k tokens (kan justeres i Meta-Header ved modelskift).

Praktiske tommelfingerregler

  • “Mange kodefiler/patches i én tråd” = hurtig stigning (10–20% hver håndfuld filer).
  • Handover-lister og lange logs = mærkbart boost (2–8%).
  • Korte Q&A uden filer = langsom stigning (0.5–1% pr. svarpar).

Handling ved grænser

  • Ved ≥70%: Vis statusbanner og tilbyd ny tråd.
  • Ved ≥85%: Auto-generér handover-pakke (links + seneste filer) og foreslå skift.
  • Ved ≥95%: Gennemfør skift straks efter at have leveret handover.

Implementeringsnote

Assistenten beregner løbende estimeret længde baseret på synlig tekst + kode i tråden og projektoverhead. Værdien rundes til nærmeste hele procent.
