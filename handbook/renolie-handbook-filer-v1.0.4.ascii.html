RENOLIE Handbook (filer) v1.0.4 (ascii)

Gyldig fra: 2025-10-26 · Tidszone: Europe/Copenhagen

0) Formaal

Kort og handlingsklar reference for RENOLIE-projektets filer (frontend/proxy). Denne version er en opdatering af v1.0.3.

1) Leveringsregler (skaerpet)

  1. Faerdige downloads: Alle filer leveres som direkte downloadbare filer. Kode maa ikke listes i traaden. Ankertekst = eksakt filnavn.
  2. ZIP=off (kun ved eksplicit anmodning). README=off (kun ved klart behov).

2) Filnavngivning (frontend/js)

  • Strengt format for js: [a-z][0-9]{3}.js — praecist ét bogstav + tre cifre (fx r101.js, f101.js).
  • Testfiler: Bruger x-serien: x001.js, x002.js, ... (kun test).
  • Ingen ?v=-parametre i fil-URL'er.

3) ASCII- og formatteringspolitik

  • Transport/lagring: ren ASCII i UI/JS/JSON/logs/Redis.
  • UI-visning: Danske bogstaver som HTML-entities (&aelig; &oslash; &aring; ...).
  • PDF: UTF-8 via indlaest font (NotoSans-URL).

4) Proxy/Render baseline

  • Build: npm install --omit=dev · Start: node renolie-proxy.mjs
  • Health: /healthz viser OK + enc-check.
  • CORS_ORIGIN: https://shop96018.sfstatic.io, https://www.renolie.dk, https://renolie.dk

5) Moduler og milepaele

M1: r101 + f101 + n101 + a101 → stabil chat-stream (ingen dobbelt output). Derefter M2 → M4.

6) Status/Traadbelastning

  • Hvert svar starter med: Traadbelastning: __ % (0–100).
  • Advar ved 70% (tidligere 80%) og foreslaa traadskifte.

7) Observability/Debug

  • UI-debugpanel: Force reload, Kill SW, Download logs/bundle, Healthz-ping, Allowlist-count.
  • RunId pr. koersel og letvaegt-debug-bundle.

8) DoD (kort)

  • Kontrakt-header, 5–8 kontrakt-tests, ingen dybe afhaengigheder, changelog ≤ 10 linjer.

9) Kanoniske links

  • Proxy health: https://renolie-proxy.onrender.com/healthz
