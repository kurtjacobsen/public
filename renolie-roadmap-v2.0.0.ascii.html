RENOLIE Roadmap v2.0.0 .ascii.html
----------------------------------
Formaal: Udvidet koereplan for Phase 2 (M5) og videre til chat-integration og PDF-motor i produktion.

============================================================
1) M5 — UI-finish og release (2-3 uger)
============================================================
Leverancer:
- Tooltips + "hjaelp?"-sektion, stoerre inputfelter, tastaturgenveje.
- Debug-panel: health ping, allowlist-count, "download log pointer".
- Autosave persona (s101 TTL 30d) paa keyup & onblur.
- Release note (≤5 bullets), PR-checkliste udfoert.

Tests:
- UI guldtest (format, stream/send, autosave).
- 429-scenarier paa /pdf/* (simuleret), korrekt fejltekst i UI.
- CSP: ingen inline scripts; alle scripts under script-src 'self'.

============================================================
2) Chat-integration (v2.1.0)
============================================================
- a101 SSE mod /chat/stream (Responses API bagved proxy). 
- n101.postJSON + onDelta callback, idempotent requestId.
- f101 formaterer deltas (b/i/link/liste); option for "hurtig" vs "noerd"-mode.

Kontrakt-tests:
- To stream-scenarier: normal + afbrudt upstream (resume).
- Ingen dobbelt-output (idem-noegle pr. request).

============================================================
3) PDF-motor (v2.1.x -> v2.2.0)
============================================================
- Implementer /pdf/make (Puppeteer) med HTML->PDF + NotoSans.
- Tilfoej /debug/fontproof og /pdf/echo til diagnostik.
- Template v1: header med titel + enc-line (æ/ø/å), liste over items, links.

Performance-maal:
- < 2 sek pr. standardopskrift
- 50–500 KB pr. PDF
- Minimer "first render" via varm browser-instans

============================================================
4) Data & Cache (v2.2.x)
============================================================
- c101: Redis TTL 300s for allowlist (konfigurerbar).
- s101: fuld CRUD for persona-profiler (default, gem/hent/slet).
- g101: rate-limit og burst control, logning via t101.

============================================================
5) Quality gate & Release (v2.3.x -> v2.4.0)
============================================================
- Endelig DoD/PR-gennemgang.
- Egen "forkromet" UI-frigivelse.
- Master-Ledger opdateret; Handover-Pack til naeste traad.

============================================================
6) Bilag: API-kontrakter (uddrag)
============================================================
/allowlist         GET -> { ok:boolean, items:[{id,name,url,price}], total:number, source:string }
/pdf/make          POST { recipe:{ title, items[], links[]? } } -> { ok:true, pdfUrl:string }
/chat/send         POST { prompt:{...}, persona? } -> { ok:true, message:string }
/chat/stream       SSE events: start, data, done; fields: requestId, tokens, cost?

Bemaerk: Endpoint-navne og kontrakter er vejledende; endelige navne bestemmes i v2.1.0 PR.
